# Contributing

Thanks for taking a look.

This repo is meant to be readable from outside the Vibecodr codebase, so improvements are welcome.

Good contributions here include:

- clarifying docs
- pointing out confusing jargon
- suggesting better reading order
- asking for more source excerpts where the current explanation feels too abstract
- flagging places where the repo still sounds too inside-baseball

If you open an issue, it helps to say which kind of question you were trying to answer when you got stuck.
